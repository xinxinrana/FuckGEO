# FuckGEO 架构设计方案 (Architecture Design)

为了实现“让所有用户看到的人工智能生成内容中没有广告”的目标，我们设计了以下三大核心模块。

## 1. FuckGEO Hub (前端展示与门户)
这是用户接触项目的第一站，也是整个生态的入口。
- **功能点**：
    - **一键复制 (Quick Copy)**：提供极简版、深度版等多种反 GEO 提示词的快速复制按钮。
    - **工具下载**：提供浏览器插件的各个版本下载（Chrome, Firefox, Edge）。
    - **实时案例展示**：对比展示“被 GEO 污染的内容” vs “净化后的内容”。
    - **用户反馈中心**：用户可以提交发现的新型 AI 广告案例。
- **技术栈建议**：Next.js + Tailwind CSS + Vercel (快速部署)。

## 2. Browser Extension (浏览器守护卫士)
这是核心的拦截工具。
- **功能点**：
    - **DOM 扫描**：实时监控主流 AI 平台的输出区域。
    - **视觉标记**：在疑似广告内容周围显示红色边框或警告图标。
    - **提示词注入**：自动在用户输入前静默注入“反 GEO 协议”。
- **技术点**：使用 Shadow DOM 确保样式不被网页原生样式干扰。

## 3. Mobile Protection (移动端方案 - 脑暴中)
由于手机端 APP 的闭源特性，我们考虑以下几种“非侵入式”方案：

### 方案 A：系统级代理 (MITM)
- **原理**：通过本地 VPN 或代理服务器，拦截 AI APP 的 HTTPS 流量，在返回前端前直接过滤掉广告字段。
- **挑战**：SSL Pinning (证书校验) 可能会导致无法解密流量。

### 方案 B：渐进式移动端防护 (Progressive Mobile Shield)
为了平衡“用户体验”与“权限门槛”，我们采取分层策略：

- **基础模式 (无需特殊权限)**：
    - **功能**：用户打开 FuckGEO App，点击“净化”按钮，自动将反 GEO 提示词复制到剪贴板。
    - **交互**：配合系统的“快捷指令”或“剪贴板监听”，让用户在跳转到 AI App 后能快速粘贴。
- **高级模式 (辅助功能自动化)**：
    - **功能**：通过 Android 辅助功能 (Accessibility Service) 实现悬浮球。
    - **自动化**：当检测到用户打开了 ChatGPT/Claude 等 App 时，悬浮球自动出现，点击后可实现“自动填入”或“屏幕实时审计”。
    - **优势**：全自动流程，一秒识别广告。

### 方案 C：FuckGEO 输入法 (Keyboard Extension)
- **原理**：用户在手机端使用 FuckGEO 专用输入法。当检测到用户在 AI APP 中输入时，自动联想或注入反 GEO 提示词。

## 4. 社区共识黑名单 (The Brain)
- **原理**：收集所有用户反馈的“广告特征”，生成动态更新的黑名单库。
- **同步**：Hub 页面、插件、移动端方案都会定期拉取该黑名单，实现全平台同步。
